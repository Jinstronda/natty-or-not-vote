- Debugging Methodology: A systematic, scientific approach to software engineering problem-solving that emphasizes:
  - Formulating clear, falsifiable hypotheses
  - Writing clean, modular test code
  - Methodical isolation and investigation of potential bug sources
  - Explicit reasoning and evidence-based debugging
  - Iterative problem-solving with user feedback
  - Adherence to professional software engineering conventions

Debugging Process Framework:
1. Hypothesis Formation
   - Create a precise, testable hypothesis about the root cause
   - Base hypothesis on observable symptoms and available evidence
   - Ensure hypothesis is specific and falsifiable

2. Test Design
   - Write clean, modular code to test the hypothesis
   - Use logging, assertions, and targeted test cases
   - Minimize side effects by isolating specific components
   - Follow TypeScript/React or Node.js best practices

3. User Testing
   - Provide clear, step-by-step testing instructions
   - Specify expected vs. actual results
   - Seek explicit user feedback

4. Outcome Analysis
   - Compare observed results to initial hypothesis
   - If hypothesis is falsified, develop a new hypothesis
   - Iterate systematically until bug is resolved

General Debugging Principles:
- Never assume without evidence
- Use explicit, testable reasoning
- Modularize and document debugging code
- Test one hypothesis at a time
- Always confirm before making permanent changes
